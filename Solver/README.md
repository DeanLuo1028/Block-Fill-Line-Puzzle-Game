# Block Fill Line Puzzle Games Solver

## 📋 項目概述

本項目實現了該遊戲的完整解題器，使用深度優先搜索（DFS）演算法自動求解。
一個用 Python 實現的「方塊填充連線謎題」遊戲**自動解題機器人**，搭配圖形化界面。
### 核心特性
- 🎯 **DFS 求解引擎**：利用深度優先搜索和回溯算法自動解題
- 🖥️ **圖形化界面**：基於 Tkinter 的互動式遊戲板
- 🏗️ **MVC 架構**：清晰的模型-視圖-控制器分離
- 📊 **自定義網格**：支援任意寬度和高度的遊戲板
- ⏯️ **步驟控制**：支援開始、暫停、重置等操作

## 🚀 快速開始

### 環境需求
- Python 3.8+
- tkinter（通常隨 Python 一起安裝）

### 安装執行

```bash
# 進入 Solver 目錄
cd Solver

# 執行程式
python Start.py
```

程式啟動後會提示輸入網格尺寸：
```
請輸入: (寬度 高度)
6 5
```

如果直接按 Enter，將使用預設值 **6×5**。

## 📖 使用方式

### GUI 操作

1. **設置遊戲板**
   - 點擊格子切換狀態：
     - 空白 (WHITE) → 牆壁 (BLACK) → 起始點 (YELLOW) → 空白

2. **解題**
   - **開始解**：啟動自動求解
   - **暫停**：暫停求解過程
   - **重置**：清空所有狀態，重新開始

### 格子狀態

| 狀態 | 顏色 | 說明 |
|------|------|------|
| SPACE | 白色 | 空白格子（待填充） |
| WALL | 黑色 | 牆壁（不可填充） |
| START | 黃色 | 起始點 |
| FILLED | 綠色 | 已填充格子 |

## 🧠 核心演算法

### DFS 求解流程

解題器使用**深度優先搜索**配合**回溯**：

```
1. 從起始點開始，標記為 FILLED
2. 嘗試四個方向（上、右、下、左）
3. 如果相鄰格子是 SPACE 且未訪問：
   - 填充該格子，標記為 FILLED
   - 遞迴搜索
4. 若所有格子都已填充 → 求解成功
5. 否則回溯，嘗試其他方向
6. 記錄死胡同，避免重複計算
```

### 主要類別

**TileState** - 格子狀態枚舉
```python
SPACE    # 空白格子
WALL     # 牆壁
START    # 起始點
FILLED   # 已填充
```

**Direction** - 方向定義
```python
UP      # ↑ (0, -1)
RIGHT   # → (1, 0)
DOWN    # ↓ (0, 1)
LEFT    # ← (-1, 0)
```

**Board** - 遊戲板管理
- 網格狀態維護
- 座標有效性檢查
- 填充狀態判定

**Solver** - 解題器
- DFS 演算法實現
- 路徑和訪問狀態記錄
- 死胡同優化
